Network Resolution Flow

Pi-hole DNS Query Path

A visual flow diagram showing how a client browser asks a local Pi-hole for a domain, how cached requests are answered immediately, and how uncached requests move upstream through DNS-over-TLS recursion before the resolved IP is returned.

Sequence

  • 1Client sends a request for a URL to the local Pi-hole.
  • 2If not cached, Pi-hole queries the upstream DoT DNS resolver.
  • 3The upstream resolver performs recursive lookup via root, TLD, and authoritative name servers.
  • 4The domain is resolved to an IP address.
  • 5The resolved IP is forwarded back to the local Pi-hole.
  • 6Pi-hole returns the final IP address to the client browser.
